# Ion Lab 3D

**전자를 직접 움직이며 깨닫는 이온의 원리** — 중학교 과학 교과 연계 3D 이온 모형 학습 웹앱.

원자번호 1~20번 원소의 최외각 전자를 마우스나 손가락으로 직접 빼고 넣으며 이온이
만들어지는 과정을 배우고, 완성한 양이온과 음이온을 결합해 전기적으로 중성인
이온 화합물을 만든다.

## 실행

빌드 도구도, 설치도, 서버도 필요 없다. `index.html`을 브라우저로 열면 끝이다.

## 학습 흐름

1. **원소 선택** — 미니 주기율표에서 1~20번 원소를 고른다.
2. **3D 전자 조작** — 최외각 전자를 끌어 전자 보관함으로 보내거나(잃음), 보관함에서
   끌어와 껍질에 넣는다(얻음). 안쪽 껍질 전자는 원자핵에 강하게 묶여 있어 움직이지 않는다.
3. **실시간 판정** — 조작할 때마다 전하량과 전자 배치가 즉시 갱신된다.
4. **상태 완성** — 안정한 전자 배치가 되면 이온식·이름·전하 계산식이 나온다.

### 학습 모드

- **자유 탐색** — 정답 판정 없이 전자를 자유롭게 넣고 빼며 전하 변화를 관찰하는 샌드박스.
- **연습** — 시스템이 목표 원소를 제시하고, 3단계 점진 힌트와 오류 진단 피드백을 제공한다.
  틀렸을 때 정답을 알려주는 대신 원인을 스스로 짚게 한다.

## 과학적 범위

- **조작 대상 (13종)** — Li, Be, N, O, F, Na, Mg, Al, P, S, Cl, K, Ca
- **예외** — H. 전자 껍질이 하나뿐이라 다른 원소와 달리 전자 8개를 기준으로 삼지 않는다.
- **비활성 기체** — He, Ne, Ar. 이미 안정하므로 조작하지 않는다.
- **공유 결합 원소** — B, C, Si. 전자를 얻거나 잃는 것만으로는 대표 이온을 설명할 수 없어
  제외하고 안내만 표시한다.
- **화합물** — 총전하 0과 최소 정수비를 모두 만족해야 완성으로 인정한다. (NaCl, MgO,
  CaCl₂, K₂O, MgCl₂, Na₂O, Al₂O₃, CaF₂ 8종이 목표)

### 용어 원칙

2022 개정 중학교 과학 교육과정에는 **'옥텟 규칙'·'듀엣 규칙'이라는 용어가 나오지 않는다.**
학생에게 보이는 문구에서는 이 용어를 쓰지 않고 "비활성 기체처럼 안정한 전자 배치"로
풀어서 설명한다. 코드의 `ruleType`은 내부 분기 키일 뿐 화면에 노출되지 않으며,
용어가 새어 나오지 않는지는 테스트로 강제한다.

마찬가지로 **중성 원자를 '이온식'이라 부르지 않는다.** 전하가 0일 때는 '원소 기호'로
표기가 바뀐다. Na는 이온이 아니기 때문이다.

> **모형의 한계** — 이 앱의 전자 껍질 모형은 전자 수를 쉽게 이해하기 위한 학습용
> 추상화다. 실제 전자는 행성처럼 정해진 원형 궤도를 돌지 않는다. 앱 안에도 이 안내가
> 상시 표시된다.

## 왜 '분자 모형'이 아니라 '결정 격자'인가

이온 화합물은 분자로 존재하지 않는다. **NaCl 분자라는 것은 없다.** 수많은 Na⁺와 Cl⁻가
정전기적 인력으로 규칙적으로 배열된 결정이 있을 뿐이고, 화학식 NaCl은 그 안의 개수비를
나타낸다. 그래서 화합물 3D 뷰어는 이온을 막대(결합선)로 잇지 않는다 — 정전기적 인력은
특정 방향으로만 작용하는 것이 아니라 주위의 모든 반대 전하를 향하기 때문이다.

1:1 화합물(NaCl, MgO, KCl, CaO, NaF, KF)의 번갈아 놓인 배열은 실제 결정 구조와 같다.
그 외의 개수비는 "개수비대로 규칙적으로 배열된다"는 것만 보여주는 학습용 모식도이며,
화면에 그렇게 명시한다.

## 퍼즐 조각 은유

양이온은 전자를 잃어 자리가 비었으므로 **홈(오목)**, 음이온은 전자가 더 붙었으므로
**돌기(볼록)** 로 그린다. 홈·돌기의 개수가 곧 전하의 크기다. 따라서 빈틈없이 맞물리는
개수를 세면 그것이 그대로 최소 정수비가 된다 — Ca²⁺의 홈 2개에 Cl⁻의 돌기 1개씩 두 개가
들어가면 CaCl₂다. 홈이 하나 비어 있으면 그것이 곧 "음이온이 하나 더 필요하다"는 신호다.

## 구조

| 파일 | 역할 |
| :--- | :--- |
| `index.html` | 화면 구조 |
| `style.css` | 스타일. 외부 폰트·CDN을 부르지 않는다 (학교 오프라인망 대비) |
| `atoms.js` | 원소 20종 · 화합물 15종 데이터 |
| `rules.js` | 판정 규칙. DOM을 모르는 순수 함수만 있어 단독으로 테스트된다 |
| `canvas-engine.js` | 원자 모형 — Canvas 2D 원근 투영 + Z-buffer 정렬로 만든 의사 3D 렌더링·입력 |
| `lattice-engine.js` | 이온 결정 격자 3D 뷰어 |
| `app.js` | 상태와 UI |
| `tests/` | 단언 483개 ([실행 방법](tests/README.md)) |
| `buyeoai/` | PRD 및 개발 계획서 |

3D 라이브러리를 쓰지 않는다. Canvas 2D 투영만으로 요구사항(원자핵 중심 고정, 최대 4개
껍질, 자유 회전·확대)을 충족하며, 보급형 태블릿 성능에는 오히려 유리하다.

## 접근성

- 색상 하나에 의존하지 않는다. 양성자(빨강·`+`), 중성자(회색), 전자(파랑·`−`)를 색상 외에
  기호·크기·테두리로도 구분한다. 최외각 전자는 흰 테두리, 내각 전자는 점선 테두리다.
- 전자 공전 애니메이션을 끌 수 있다 (모션 멀미 방지). `prefers-reduced-motion`도 존중한다.
- 데스크톱 마우스와 모바일 터치를 모두 지원한다.
- 데스크톱 3분할 → 태블릿 2분할 → 모바일 세로 스택 반응형.
